# Архитектурный обзор

Этот файл нужен как технологически-нейтральный источник контекста для моделей.

## Что здесь описывать

- цели системы;
- ключевые доменные сущности;
- границы подсистем;
- основные сценарии и потоки данных;
- архитектурные инварианты;
- нефункциональные требования.

## Что не писать сюда

- версии библиотек;
- частные реализации конкретного фреймворк-слоя;
- команды сборки;
- детали тестовых инструментов.
