# Дорожная карта

## Видение

Сделать шаблонный репозиторий для Cursor устойчивым переиспользуемым каркасом для новых проектов, в котором правила, документация и процессы развития контекста управляются как отдельный инженерный продукт.

## Цель v1.0

Подготовить минимально зрелую версию репозитория, которая:
- задаёт стабильный базовый набор глобальных правил;
- поддерживает управляемое развитие локальных правил;
- содержит понятную инициализацию нового проекта;
- описывает единый процесс добавления, проверки и оценки правил.

## Границы v1.0

### Включает
- базовые глобальные правила;
- архитектурный базовый каркас в `docs`;
- шаблоны компонентной документации;
- процессы инициализации и цикла изменения правил;
- bash- и PowerShell-скрипты инициализации;
- журнал изменений и дисциплина выпусков.

### Не включает
- большой каталог стек-специфичных правил для всех возможных технологий;
- автоматические CI-пайплайны для проверки качества правил;
- сложная метрика качества с автоматической оценкой;
- настройки под конкретную модель или провайдера;
- полноценный каталог примеров для разных типов репозиториев.

## Этапы

### ☑ M1 — Стабилизация базового каркаса
- привести глобальные правила к единому стилю;
- проверить правила именования;
- выровнять README и архитектурную документацию;
- убедиться, что bash- и PowerShell-инициализация работают.

### ☑ M2 — Управление изменениями правил
- зафиксировать единый процесс добавления, проверки и оценки правил;
- ввести шаблон предложения изменения правил;
- определить условия удаления, разделения и устаревания правил.

### ☐ M3 — Оценка на реальных кейсах
- определить набор эталонных задач;
- зафиксировать таксономию типовых ошибок;
- использовать простой процесс сравнения до/после изменения правил.

### ☐ M4 — Выпуск v1.0
- обновить журнал изменений;
- сделать финальную чистку структуры;
- отметить стабильный базовый каркас для использования в новых проектах.

## После v1.0
- примеры для Python-ориентированных, фронтенд-ориентированных и ориентированных на платформу данных репозиториев;
- отдельные правила `1x-*` для тестирования, общих контрактов и документов архитектурных решений;
- опциональные CI-проверки согласованности шаблона;
- политика семантического версионирования изменений системы правил.
