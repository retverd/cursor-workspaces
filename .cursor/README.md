# Правила Cursor

Папка `.cursor/rules/` содержит базовые инструкции для агента. Правила должны быть короткими, прикладными и разделёнными по ответственности.

## Правила именования

- `00-agent-core.mdc` — базовое поведение агента;
- `01-rules-generation.mdc` — правила развития самой системы правил;
- `02-architecture-principles.mdc` — архитектурные принципы и контрактные риски;
- `03-repo-operations.mdc` — файловые операции, миграции и рискованные изменения;
- `04-docs-concepts.mdc` — локальное правило для документации в `docs`;
- `1x-*.mdc.example` — примеры компонентных правил;
- `8x-*.mdc.example` — примеры временных и миграционных правил;
- `90-verification.mdc` — обязательная проверка результата;

## Принципы

- `alwaysApply: true` используйте только для устойчивых глобальных правил.
- Технические детали и специфические правила для конкретной технологии должны подключаться через `globs`.
- Новые правила добавляются под повторяющиеся типовые ошибки, а не под единичные задачи.
- Документация объясняет контекст; правила задают действия агента.
