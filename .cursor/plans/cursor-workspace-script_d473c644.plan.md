# План создания скрипта

- Создать файл [`list_cursor_workspaces.py`](list_cursor_workspaces.py) в корне репозитория.
- В скрипте определить два кандидата для `workspaceStorage`:
  - `%APPDATA%\Cursor\User\workspaceStorage`, вычисленный для текущего пользователя через переменную окружения `APPDATA`
  - `C:\ProgramData\Cursor\User\workspaceStorage`
- Выбрать первый существующий каталог из списка; если ни один не существует, вывести понятное сообщение об ошибке и завершиться с ненулевым кодом.
- Пройти только по непосредственным подпапкам выбранного каталога.
- Для каждой подпапки читать только файл `workspace.json`, если он существует; другие файлы и вложенные каталоги не анализировать.
- Извлекать из JSON поле `folder`, декодировать `file:///...` URI в обычный путь Windows через стандартные библиотеки Python (`urllib.parse`, `pathlib`), а не строковыми заменами вручную.
- Для удалённых URI вроде `vscode-remote://...` оставить исходное значение, так как это не локальный `file URI` Windows.
- Для каждого расшифрованного локального пути проверить, существует ли такая папка на диске.
- Для удалённых и прочих нелокальных URI статус существования выводить как `n`, так как это не локальные папки на диске.
- Вывести строки в формате `имя_подпапки -> e/n -> путь`, где `e` означает, что папка существует, а `n` означает, что папка не существует.
- После создания выполнить краткую проверку: запустить скрипт, убедиться что вывод соответствует ожидаемому формату со статусом существования, и проверить diff.
