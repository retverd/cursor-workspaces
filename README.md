# Рабочие пространства Cursor

Проект предназначен для локальных операций с рабочими пространствами Cursor, которые не реализованы в базовом интерфейсе или требуют аккуратной работы с пользовательскими данными: анализа содержимого рабочих пространств, проверки связей между проектами и подготовки операций обслуживания.

## Назначение

Проект помогает работать с локальными рабочими пространствами Cursor как с отдельным набором данных, который можно анализировать, проверять и при необходимости преобразовывать.

Типовые сценарии:

- анализ содержимого рабочих пространств Cursor;
- поиск связей между проектами, путями, рабочими пространствами и чатами;
- перенос чатов при переименовании или перемещении проектов;
- подготовка локальных операций, которых нет в стандартном интерфейсе Cursor;
- документирование ограничений и рисков при работе с пользовательскими данными.

## Скрипты

### `list_cursor_workspaces.py`

Скрипт на Python 3.14 выводит список рабочих пространств Cursor из локального `workspaceStorage`.

При запуске он выбирает первый существующий каталог:

- `%APPDATA%\Cursor\User\workspaceStorage`, вычисленный для текущего пользователя через переменную окружения `APPDATA`;
- `C:\ProgramData\Cursor\User\workspaceStorage`.

Скрипт проходит только по непосредственным подпапкам выбранного каталога и читает только файл `workspace.json` в каждой подпапке. Остальные файлы и вложенные каталоги не анализируются.

Для поля `folder` из `workspace.json` скрипт:

- декодирует локальные `file:///...` URI в обычные Windows-пути;
- оставляет удалённые и прочие нелокальные URI без изменения;
- проверяет, существует ли расшифрованная папка на диске.

Формат вывода:

```text
<имя_подпапки> -> e/n -> <путь>
```

Где `e` означает, что папка существует, а `n` означает, что папка не существует. Для удалённых URI, например `vscode-remote://...`, статус выводится как `n`, так как это не локальная папка на диске.

Запуск:

```powershell
py -3.14 .\list_cursor_workspaces.py
```

## Принципы

- Все операции выполняются локально и должны уважать данные пользователя;
- Чтение и анализ предпочтительнее записи, если задача не требует изменения данных;
- Потенциально опасные действия должны быть явными и проверяемыми;
- Логика работы с рабочими пространствами должна быть отделена от конкретного интерфейса запуска.

## Текущий статус

В проекте реализован первый скрипт только для чтения для инвентаризации локальных рабочих пространств Cursor. Перед реализацией операций, которые изменяют данные рабочих пространств Cursor, нужно явно определить формат входных данных, сценарий проверки результата и требования к резервному копированию или откату.
