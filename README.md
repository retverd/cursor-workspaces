# Cursor Starter Kit

Шаблон для запуска новых проектов в Cursor с управляемым контекстом агента: короткие глобальные правила, локальные правила для компонентов и документация как источник архитектурного смысла.

## Структура

- `.cursor/rules/00-agent-core.mdc` — базовое поведение агента;
- `.cursor/rules/01-rules-generation.mdc` — правила развития самой системы правил;
- `.cursor/rules/02-architecture-principles.mdc` — архитектурные принципы и контрактные риски;
- `.cursor/rules/03-repo-operations.mdc` — файловые операции, миграции и рискованные изменения;
- `.cursor/rules/04-docs-concepts.mdc` — локальное правило для документации в `docs`;
- `.cursor/rules/1x-*.mdc.example` — примеры компонентных правил;
- `.cursor/rules/8x-*.mdc.example` — примеры временных и миграционных правил;
- `.cursor/rules/90-verification.mdc` — обязательная проверка результата;
- `docs/architecture/` — концепция, обзор и архитектурные решения;
- `docs/components/` — шаблоны контекста для компонентов;
- `docs/workflows/` — процессы инициализации и изменения правил;
- `docs/templates/` — шаблоны документов и предложений изменений;
- `scripts/bootstrap.sh` и `scripts/bootstrap.ps1` — перенос базового каркаса в новый проект.

## Как использовать

1. Создайте новый проект из шаблона или запустите скрипт инициализации для существующего проекта.
2. Заполните `docs/architecture/overview.md` и релевантные файлы в `docs/components/`.
3. Уточните `globs` в локальных правилах под структуру репозитория.
4. Удалите примеры правил и документов, которые не относятся к проекту.
5. Добавляйте новые локальные правила только под повторяющиеся типовые ошибки.

## Принцип базового каркаса

**Правила** задают операционные инструкции для агента.

**Документация** объясняет цели, границы, решения и компромиссы.

**Глобальный контекст** должен оставаться коротким; технические детали должны жить в локальных правилах или компонентной документации.
